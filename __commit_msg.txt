fix(dms): scope renewal replacement search by owning entity, relabel superseded status, trim documents table columns

Renewal replacement bugfix:
- searchDmsDocumentsForRenewalReplacement previously scoped candidates
  by document_type_id only, so e.g. every employee's Emirates ID/Passport
  showed up as a valid replacement for one specific employee's renewal.
- Now resolves the original document's owner via dms_document_links
  (employee, party, vehicle, etc.) and restricts candidates to documents
  linked to that same entity. Falls back to type-only scoping for
  documents with no entity link (generic/company-wide documents).
- DmsCompleteRenewalDialog now shows which entity the search is scoped
  to and reflects it in the empty-state message.

Status label:
- "Superseded" document status now displays as "Renewed" everywhere
  (badge, filter dropdown, expiry banner, complete-renewal dialog/toast).
  Underlying DB value/status code is unchanged.

Documents table:
- Removed the Conf. and Created columns to declutter the list; widened
  remaining resizable columns to use the freed space.
