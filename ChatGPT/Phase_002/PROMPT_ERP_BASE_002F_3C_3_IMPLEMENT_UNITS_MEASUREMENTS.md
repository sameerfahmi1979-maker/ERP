## Critical Corrections From Sameer / Dina Review

Before implementation, apply these corrections strictly:

### 1. Do Not Add UOM_ROUNDING_METHODS Now

The planning file evaluated `UOM_ROUNDING_METHODS` as optional.

Decision:

```text
DO NOT implement UOM_ROUNDING_METHODS in this phase.
```

Reason:

```text
No rounding engine exists yet.
No conversion calculation engine exists yet.
Adding rounding lookup now creates unused complexity.
```

### 2. Implement uom_conversions Table, But Seed It Empty

The `uom_conversions` table must be created for future readiness.

However:

```text
Do not seed direct conversions like M3 → L or KG → G in uom_conversions.
```

Normal same-category conversions are already handled by:

```text
units_of_measure.conversion_factor_to_base
```

The `uom_conversions` table is reserved for:

```text
special conversions
cross-category conversions if approved later
legacy exceptions
future item-specific conversions
future packaging conversions
```

### 3. MONTH Is Approximate

The TIME category may include `MONTH`, but `MONTH = 730 HOUR` is approximate.

Implement:

```text
MONTH conversion_factor_to_base = 730
```

But add clear notes:

```text
Approximate monthly unit. Exact month billing/rental logic must be handled in future billing/rental modules.
```

Do not make future billing depend blindly on this approximation.

### 4. Packaging Conversions Are Non-Universal

PACKAGING category is allowed, but packaging conversions must be treated carefully.

Implement packaging units:

```text
EA
BOX
BAG
BUNDLE
ROLL
DRUM
PALLET
CONTAINER
```

But document clearly:

```text
PACKAGING conversions are not universal.
1 BOX is not always equal to a fixed number of EA.
Item-specific packaging conversions must be handled later in Inventory Item Master.
```

Do not seed universal BOX → EA, BAG → EA, PALLET → EA conversions.

### 5. Strict Future UOM Reuse Rule

Include in code comments/report:

```text
Future modules must not hardcode KG, TON, M, CM, GAL, BOX, EA, DAY, HOUR, etc.
Future modules must use UnitOfMeasureSelect, UnitByCategorySelect, or approved UOM server actions.
```

This applies to:

```text
Inventory
Procurement
Fleet
Workshop
Weighbridge
Fuel Management
CRM
Billing
Scrap/Waste/Demolition
Equipment Rental
```

## 4.3 units_of_measure

Create table:

```text
units_of_measure
```

Fields:

```text
uom_category_id bigint references public.uom_categories(id) on delete restrict
unit_code text not null
unit_name_en text not null
unit_name_ar text
symbol text
conversion_factor_to_base numeric(20,10) not null default 1
is_base_unit boolean not null default false
decimal_places integer not null default 2
allow_fraction boolean not null default true
description_en text
description_ar text
notes text
standard system/locked/active/audit fields
```

Constraints:

```text
unique(uom_category_id, unit_code)
unit_code uppercase
unit_code only A-Z, 0-9, underscore
conversion_factor_to_base > 0
decimal_places between 0 and 6
one base unit per category using partial unique index
```

Required unique index:

```sql
create unique index if not exists idx_units_one_base_per_category
on public.units_of_measure (uom_category_id)
where is_base_unit = true;
```

## 4.4 uom_conversions

Create table:

```text
uom_conversions
```

Fields:

```text
from_uom_id bigint references public.units_of_measure(id) on delete restrict
to_uom_id bigint references public.units_of_measure(id) on delete restrict
conversion_factor numeric(20,10) not null
conversion_formula_code text
is_bidirectional boolean not null default false
notes text
standard system/locked/active/audit fields
```

Constraints:

```text
unique(from_uom_id, to_uom_id)
from_uom_id <> to_uom_id
conversion_factor > 0
```

Important:

```text
Create this table but do not seed it initially.
```

Add comment:

```text
Reserved for special, future, cross-category, legacy exception, or item-specific conversions. Normal same-category conversion uses units_of_measure.conversion_factor_to_base.
```
