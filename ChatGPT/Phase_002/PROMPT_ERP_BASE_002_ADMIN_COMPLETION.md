## 3. Critical Problem to Fix First

The browser console currently shows this error on the Users page:

```text
Could not embed because more than one relationship was found for 'user_profiles' and 'user_roles'
```

This is likely caused by an ambiguous Supabase/PostgREST relationship embed between `user_profiles` and `user_roles`.

### Required fix

Fix the user list query immediately before building more features.

Acceptable approaches:

1. Use explicit foreign key relationship names if available.
2. Split the user list query into multiple clear queries:
   - fetch `user_profiles`
   - fetch `user_roles`
   - fetch `roles`
   - fetch `owner_companies`
   - fetch `branches`
   - merge on server side
3. Create a safe SQL view or RPC only if needed and only after reviewing RLS/security implications.

Preferred approach for Phase 002:

Use server-side queries that avoid ambiguous PostgREST embeds and preserve RLS.

Do not weaken RLS.

Do not create broad service-role list queries unless explicitly documented and restricted to server-only admin functions.

1. Use server-side permission checks.
2. Respect RLS.

7. Write audit logs.
8. Handle Supabase errors clearly.
9. Avoid leaking sensitive details to client.
10. Avoid using client-side-only checks for security.

- Use only server-only files.
- Validate current user is authorized before using service-role.
- Never use service-role for normal user-scoped data unless absolutely required.
- Prefer RLS-safe user client where possible.
- Document each service-role usage.
