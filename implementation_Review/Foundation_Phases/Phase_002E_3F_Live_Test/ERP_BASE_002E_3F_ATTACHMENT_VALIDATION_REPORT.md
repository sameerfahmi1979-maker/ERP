## 🔍 Known Attachment Considerations

### Base64 Encoding Overhead

**Issue**: Email attachments are base64-encoded, which increases size by ~33%.

**Example**:
- Original file: 100 KB
- Base64 encoded: ~133 KB
- Actual size sent via Graph API: 133 KB

**Impact**:
- If `MICROSOFT_MAIL_MAX_ATTACHMENT_MB=10`, actual file limit is ~7.5 MB
- Phase 002E.3D accounts for this (uses original file size for validation)

### PDF Font Embedding

**Issue**: PDFs embed fonts for consistent rendering, increasing file size.

**Mitigation**:
- Phase 002E.3B uses standard fonts (Arial, Times New Roman)
- These fonts are widely available, reducing embedding overhead
- If custom fonts needed, file size may increase significantly

### CSV UTF-8 BOM for Excel

**Issue**: Excel does not auto-detect UTF-8 without BOM (Byte Order Mark).

**Mitigation**:
- Phase 002E.3B prepends UTF-8 BOM (`\uFEFF`) to all CSV files
- This ensures Arabic/special characters display correctly in Excel
- Text editors may show BOM as weird character (expected, harmless)
