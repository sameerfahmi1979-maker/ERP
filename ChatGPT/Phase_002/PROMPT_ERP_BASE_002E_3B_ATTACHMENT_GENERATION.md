## 1. Purpose

The ERP already supports downloading/exporting:

- CSV
- Excel
- PDF
- Print

But email requires file attachments in this structure:

```ts
{
  filename: string
  contentType: string
  base64Content: string
  sizeBytes: number
}
```

This phase must create attachment-generation functions that reuse the existing export engine logic, but return in-memory base64 attachments instead of triggering browser downloads.

## 7. CSV Attachment Requirements

Create `generateCSVAttachment(options)`.

It must:

1. Reuse the same CSV formatting rules as `exportToCSV`.
2. Use UTF-8 BOM if current CSV download uses it.
3. Escape commas, quotes, and newlines correctly.
4. Support Arabic/Unicode text.
5. Convert content to base64 using server/browser-safe utility.
6. Return:
   - filename
   - contentType
   - base64Content
   - sizeBytes

Use:

```ts
stringToBase64Utf8()
```

from email attachment utilities if available.

---

## 8. Excel Attachment Requirements

Create `generateExcelAttachment(options)`.

It must:

1. Reuse the same workbook/worksheet structure as `exportToExcel`.
2. Include metadata if existing Excel export includes metadata:
   - title
   - subtitle
   - generated by
   - generated at
   - filters if any
3. Include headers and data rows.
4. Preserve column widths where practical.
5. Generate XLSX ArrayBuffer.
6. Convert ArrayBuffer to base64.
7. Return correct MIME type and size.

Use:

```ts
arrayBufferToBase64()
```

from email attachment utilities if available.

Do not trigger file download.

---

## 9. PDF Attachment Requirements

Create `generatePDFAttachment(options)`.

It must:

1. Reuse the same PDF layout as `exportToPDF`.
2. Include title, subtitle, generated by, generated at, filters, table, footer, page numbers.
3. Use same orientation logic as download PDF.
4. Generate PDF ArrayBuffer.
5. Convert ArrayBuffer to base64.
6. Return correct MIME type and size.

Do not trigger file download.

## 11. Type and Runtime Compatibility

The attachment functions may be called from client UI later and possibly from server action later.

Ensure utilities are compatible or clearly document environment constraints.

Important:

- `btoa` may not exist in Node depending runtime.
- If existing `attachment-utils.ts` uses browser-only APIs, adjust it to support Node and browser safely.
- In Node, use `Buffer.from(...).toString("base64")` if available.
- In browser, use `btoa` as fallback.

Do not import Node-only modules into client components if these functions will be used client-side.

Document whether attachment generation is intended for client, server, or both.

Recommended approach:

- Implement base64 helpers that work in both browser and Node where possible.
- Avoid direct `fs`, path, or Node-only APIs.
