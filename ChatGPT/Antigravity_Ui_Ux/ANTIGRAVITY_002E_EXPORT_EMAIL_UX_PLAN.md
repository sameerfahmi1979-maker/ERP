# ANTIGRAVITY_002E_EXPORT_EMAIL_UX_PLAN — Global Export & Email Sharing Architecture

This document plans the UX flow, toolbar placement, layout standards, and user-facing notifications for exporting reports and emailing transaction records directly from the ERP.

---

## 1. Export Action Placement & Layouts

Every list view (e.g., branches, organizations, audit logs) and detail view drawer must include a standard **Share/Export Menu**.

### A. List View Placement
In the data tables (such as `src/components/erp/data-toolbar.tsx`), a dropdown button labeled **Export** will sit in the top-right toolbar.
- **Export Page**: Downloads only the currently visible data rows in the active viewport.
- **Export All Filtered**: Triggers a client-side or server-side export containing all matching records across all pages, respecting current search queries and filter parameters.
- **File Options**: Excel (.xlsx), CSV (.csv), and PDF (.pdf).

### B. Detail / Drawer View Placement
In the sticky header of details views, a **More Actions** menu icon (vertical dots) exposes:
- **Print**: Opens the system print utility with custom print stylesheets.
- **PDF Download**: Generates a high-quality formatted PDF profile.
- **Email Document**: Launches the integrated email composition sheet.

## 2. Enterprise PDF & Print Template Standard

To maintain enterprise-grade presentation, all exports (PDF and Print Layouts) must follow a unified structure:

```text
┌────────────────────────────────────────────────────────┐
│  [Company Logo]                 ALLIANCE GULF GROUP    │
│  Alliance Gulf Transport        Branch: Abu Dhabi HQ   │
│  TRN: 100452390100003           Tel: +971 2 555 1234   │
├────────────────────────────────────────────────────────┤
│                                                        │
│                  REPORT / INVOICE TITLE                │
│                  Reference: EMP-2026-0042              │
│                                                        │
│  Generated By: Sameer Fahmi       Date: 2026-05-27     │
│  Filters: [Branch: Abu Dhabi] [Status: Active]         │
│                                                        │
├────────────────────────────────────────────────────────┤
│  Data Table / Form Fields Grid                         │
│  ...                                                   │
├────────────────────────────────────────────────────────┤
│  [Stamp Space]                          [Authorized]   │
│                                          Signature     │
│  Page 1 of 3                                           │
└────────────────────────────────────────────────────────┘
```

- **TRN & Trade License Display**: Mandatory for UAE-based company branches. If the parent company is selected, its tax registration details are rendered in the top-left metadata block.
- **Arabic Translation**: Report headers must include both English and Arabic titles where possible (e.g. *Alliance Gulf Transport & Construction LLC / تحالف الخليج للنقل والمقاولات ذ.م.م*).
- **Metadata**: Every report must print `Generated By: {User_Email}` and `{Timestamp}` to track access and audit trails.

## 3. Email Composition Modal Details

Selecting **Send by Email** triggers an overlay modal containing:
- **From Address**: Read-only field displaying the system's authorized outbound email address (e.g., `noreply@alliancegulf.ae`) or the current user's profile alias.
- **Recipients**: Inputs for `To`, `CC`, and `BCC` with autocomplete searching across existing system users.
- **Subject**: Pre-filled template subject (e.g., `[Alliance Gulf ERP] Profile Update: Alliance Gulf Transport L.L.C`).
- **Template Selector**: Dropdown to select pre-configured email message bodies (e.g., "Standard Audit Report", "Record Transfer Notice").
- **Attachment Preview**: A small card indicating the generated file name (e.g., `Alliance_Gulf_Transport_Profile.pdf`) with its size, allowing the user to view a quick preview before sending.

---

## 4. Email Transaction States & UX Transitions

The interface handles the asynchronous email dispatch with 5 visual phases:

1. **Preparing Attachment**: A progress bar showing PDF generation progress.
2. **Sending Outbound**: The "Send" button is replaced by a disabled spinner (`Loader2`) reading `Sending email...`.
3. **Success State**: The modal slides closed and a success toast notifications shows `Email sent to info@alliancegulf.ae with attachment.`.
4. **Failure State**: The modal remains open, a warning banner displays the error details (e.g., `SMTP Connection Timeout`), and the action button changes to `Retry Send`.
5. **Draft Cache**: If closed during editing, the system prompts the user to save as an email draft or discard.
