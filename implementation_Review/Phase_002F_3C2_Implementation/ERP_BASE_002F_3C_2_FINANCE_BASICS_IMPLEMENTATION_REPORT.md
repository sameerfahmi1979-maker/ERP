No `auth.users.id` references.

## 12. RLS Policies

24 policies (4 × 6 tables):

- **SELECT:** active records OR `current_user_has_permission('master_data.finance_basics.view')`
- **INSERT:** `master_data.finance_basics.manage`
- **UPDATE:** manage + (not locked OR `current_user_has_role('system_admin')`)
- **DELETE:** system_admin via `user_roles` subquery + `current_user_profile_id()`

## 22. Known Limitations

1. **Migration history:** Remote DB applied via MCP chunks; local `supabase db push` requires migration history repair.
2. **Branches geography (002F.3C.1B.2):** `branch_id` on cost/profit centers is nullable until branches FK migration completes.
3. **Parent hierarchy:** Self-parent blocked; full descendant cycle prevention deferred to future enhancement.
4. **Export server actions:** Table export uses `ERPDataTable` client export; dedicated `exportX()` server actions not implemented (not required for phase completion).
5. **Org/branch selects on cost/profit forms:** Owner company and branch use numeric ID inputs; dedicated org/branch select components can be added when those modules expose selects.
6. **Manual browser testing:** Not executed in this session.
