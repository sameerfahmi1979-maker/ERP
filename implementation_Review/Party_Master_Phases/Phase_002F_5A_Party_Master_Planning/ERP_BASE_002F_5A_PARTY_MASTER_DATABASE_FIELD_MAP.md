## Legend

| Symbol | Meaning |
|---|---|
| PK | Primary key (`bigint generated by default as identity`) |
| FK → table | Foreign key reference |
| UQ | Unique constraint |
| IDX | Non-unique index |
| PUQ | Partial unique index |
| RLS | Row Level Security enabled |
| AUD | Standard audit: `created_at`, `created_by`, `updated_at`, `updated_by` |
| SYS | System flags: `is_active`, `is_locked`, `is_system` where applicable |

**Standard audit block (all transactional tables):**

```sql
created_at   timestamptz NOT NULL DEFAULT now()
created_by   bigint REFERENCES user_profiles(id)
updated_at   timestamptz NOT NULL DEFAULT now()
updated_by   bigint REFERENCES user_profiles(id)
```

**Standard lookup master block:**

```sql
id bigint PK, code text UQ, name_en text, name_ar text,
description text, is_system boolean DEFAULT false, is_active boolean DEFAULT true,
sort_order integer DEFAULT 0, + AUD
```

## 5. Indexes Summary

See draft SQL for `CREATE INDEX` statements. Key partial unique indexes:

```sql
-- One primary type assignment per party
CREATE UNIQUE INDEX uq_party_type_assignments_primary
  ON party_type_assignments (party_id) WHERE is_primary AND is_active;

-- No duplicate active type
CREATE UNIQUE INDEX uq_party_type_assignments_active
  ON party_type_assignments (party_id, party_type_id) WHERE is_active;

-- Similar for is_primary on licenses, contacts, addresses, bank_details, tax_registrations
-- Active TRN unique
CREATE UNIQUE INDEX uq_party_tax_trn_active
  ON party_tax_registrations (tax_registration_number)
  WHERE is_active AND tax_registration_number IS NOT NULL;

-- Active license number unique (global)
CREATE UNIQUE INDEX uq_party_license_number_active
  ON party_licenses (license_number) WHERE is_active AND license_number IS NOT NULL;
```
